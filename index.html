File Icon Library: An Open-source, CSS & JavaScript Rendered Icon Library
Alpha 1
What is the File Icon Library?
The File Icon Library is an open-source library of icons rendered using only CSS and JavaScript. There are no image files used in any of the icons, thus minimizing the loading time of a webpage.
What browsers support the File Icon Library?
As this is an ALPHA release, we cannot guarantee 100% compatibility with all browsers. The browsers we have tested in that support the File Icon Library include:
  • Chrome 36 - 100% - all features are supported
  • Safari 8 - 95% - doesn’t support document.registerElement, but this doesn’t affect rendering
  • Internet Explorer 11.0.9 - 95% - doesn’t support document.registerElement, but this doesn’t affect rendering
  • Firefox 31 - 90% - has issues rendering certain icons (MS Office Icons)
  • Opera 12.16 - 80% - has issues with icon layout

The Code
A custom HTML tag <file-icon></file-icon> is used to render file icons.

Parameters:
(note: < > required parameter, [ ] optional parameter, { } set by engine)
  • data-grid: [boolean] - overlays a grid, used for icon design and alignment
  • data-label: [label] - will default to subtype label if available
  • data-rendered: {boolean} - set by the render engine
  • data-size: <xlarge>|<large>|<medium>|<small>|<tiny>|<micro>
  • data-type: <type [subtype]> type list
Demo - Preset Type & Subtype
Code:
<file-icon data-type="file pdf" data-size="medium"></file-icon>
